# Attendance Management System - User Manual

## 1. Introduction
This manual is for faculty-side and student-side users (including class teacher and coordinators where UI overlap exists).

## 2. Getting Started
1. Open application URL.
2. Log in with institutional credentials.
3. You will be redirected to your role dashboard.
4. Use left navigation/top menus to access modules.

## 3. Common Navigation
- **Dashboard**: Overall stats and shortcuts.
- **Profile**: Manage personal profile information.
- **Notifications**: Read updates and alerts.
- **Change Password**: Update account password.

## 4. Student Guide
### Available Functions
- View today attendance and cumulative summary.
- View subject-wise attendance details.
- View timetable.
- View notices/notifications.
- View leave history.

### Recommended Daily Workflow
1. Login → Student Dashboard.
2. Verify today attendance.
3. Check subject-wise percentage weekly.
4. Read notifications and class notices.

## 5. Faculty Guide
### Available Functions
- Mark attendance for assigned classes.
- View class attendance and summaries.
- Request/edit attendance correction within policy window.
- Generate role-permitted reports.
- Use QR attendance (if enabled for class).

### Attendance Marking Steps
1. Open attendance module.
2. Select subject/date/session/period.
3. Mark status (`P`, `A`, `L`, `ML`) for each student.
4. Submit and verify success response.

## 6. Class Teacher Guide
- Manage class student roster.
- Send notices to class cohorts.
- View leave requests.
- Trigger absent/monthly alerts.
- Download monthly/semester reports.

## 7. Attendance Coordinator Guide
- Monitor department classes.
- Analyze below-threshold/above-threshold lists.
- Download reports by class/semester/month.
- Push coordinator alerts.

## 8. Time Table Coordinator Guide
- Manage timetable entries.
- Update schedule slots and assignments.
- Coordinate with HOD/Admin for schedule conflicts.

## 9. Best Practices
- Mark attendance as close to live session time as possible.
- Include remarks for non-standard entries.
- Avoid shared credentials.
- Re-check low-attendance reports before export/share.

## 10. FAQ (End User)
**Q: I am redirected to login repeatedly.**  
A: Session token may have expired; logout and login again.

**Q: I cannot see my page after login.**  
A: Your role may not have access; contact Admin/HOD.

**Q: Attendance status looks incorrect.**  
A: Raise correction through faculty/class teacher/HOD workflow.
